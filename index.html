Quick start

Two ways to use SPUX. Pick whichever fits your workflow.

1. One-liner installer

Bootstrap VSCode + Cline + the spux CLI in a single command.

curl -sL get.agentspux.ai/init | bash

# then run any tool by alias
spux run proxy
spux run vscode-setup
spux run llm-connect

2. Clone and run locally

Read every script before you run it. That's the whole point.

git clone https://github.com/agentspux/agentspux.github.io.git
cd agentspux.github.io
bash spux-init.sh
bash core/engine.sh status
bash core/validate.sh scripts/*.sh

Architecture

Three small ideas, composed.

📒 Registry

A flat registry.json maps short aliases → script paths. A redirect worker resolves get.agentspux.ai/<id> to a raw GitHub URL.

🧾 SPUX Manifest

Every script declares its ID, title, author, platform, and description in a header block. That metadata powers this page.

🛡️ Safety

Automated regex audits + AI logic review on every PR. sudo always prompts. System-directory writes always prompt. MIT-licensed end to end.

🔌 A2A & MCP

YAML specs for Agent-to-Agent handshakes (specs/a2a/) and pluggable Model Context Protocol configs (specs/mcp/).

Contribute a script

If you've automated something, share it. CI runs the security audit — if it passes, your script gets a registry alias.

# 1. Drop your script in community/<your-handle>/
# 2. Add the SPUX_MANIFEST header
# 3. Open a PR — automated audit runs immediately
# 4. Once merged, your script gets a get.agentspux.ai/<alias> route
